## 2024-05-18 - Cryptographically Secure Random Number Generation
**Vulnerability:** Weak random number generation (`Get-Random`) used for creating new Active Directory user passwords.
**Learning:** `Get-Random` in PowerShell 5.1 relies on the non-cryptographically secure `System.Random` class, meaning that the generated passwords could theoretically be predicted by a malicious actor who observes the PRNG's outputs.
**Prevention:** Use `System.Security.Cryptography.RandomNumberGenerator::Create()` for generating strong, unpredictable random numbers for security-critical tasks like password generation, tokens, or encryption keys.

## 2025-04-03 - [Cross-Site Scripting (XSS) in HTML Dashboard Generation]
**Vulnerability:** The HTML dashboard generated by `GenDash.ps1` was vulnerable to DOM-based XSS. Unsanitized data dynamically retrieved from Active Directory and log files (such as usernames, operator names, and log details) were being injected directly into the DOM via `.innerHTML` string interpolation (e.g., `<td ...>${log.username}</td>`).
**Learning:** Any user-controlled or external data, even from trusted internal systems like AD or local log files, must be treated as untrusted when rendered into HTML. Injecting it directly allows an attacker to create malicious accounts or log entries that execute arbitrary JavaScript when the dashboard is viewed.
**Prevention:** Always sanitize or escape dynamically injected data when using `.innerHTML`. Added a custom `escapeHtml` JavaScript function that replaces dangerous characters (`&`, `<`, `>`, `"`, `'`) with their safe HTML entity equivalents before rendering.
