## 2024-05-18 - Cryptographically Secure Random Number Generation
**Vulnerability:** Weak random number generation (`Get-Random`) used for creating new Active Directory user passwords.
**Learning:** `Get-Random` in PowerShell 5.1 relies on the non-cryptographically secure `System.Random` class, meaning that the generated passwords could theoretically be predicted by a malicious actor who observes the PRNG's outputs.
**Prevention:** Use `System.Security.Cryptography.RandomNumberGenerator::Create()` for generating strong, unpredictable random numbers for security-critical tasks like password generation, tokens, or encryption keys.
## 2024-05-24 - Command Injection via Start-Process -Command
**Vulnerability:** Untrusted user input (`$targetId`) was string-replaced manually (`Replace("'", "''")`) and then injected directly into a script block passed to `Start-Process powershell.exe -Command`, opening up the potential for remote command injection if edge-case escape sequences are used.
**Learning:** Manual string replacement is never sufficient to protect against command injection when dynamically constructing a script block that will be executed by another instance of PowerShell. Single quotes, backticks, and other escape mechanisms can be abused to break out of the string boundary.
**Prevention:** To prevent command injection when spawning child `powershell.exe` processes that require user-controlled arguments, use `-EncodedCommand` with a Base64-encoded UTF-16LE string. For maximum security, Base64 encode the user-controlled variable separately and decode it within the child process script to ensure it is handled strictly as data.
