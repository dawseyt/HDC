Introduction

Welcome to HelpDesk Companion — the IT team's central tool for managing Active Directory users, remote computers, printers, and helpdesk ticketing from a single, fast interface.

Account Unlocks
Unlock individual or all accounts instantly, with optional auto-unlock automation.
Password Resets
Reset AD passwords with built-in complexity enforcement and a strong password generator.
Remote Computer Tools
Processes, services, software, devices, event logs, disk space — all remotely accessible.
Printer Manager
Install, remove, and manage remote printers including driver deployment.
Freshservice Integration
Create tickets, open user records, and look up assigned computers without leaving the app.
Audit Logging
Every action is logged with timestamp, technician, and target to a shared CSV on the network.
Group Management
View current AD group memberships and add or remove groups without leaving the app.
Ticket Feed
See a user's last 8 Freshservice tickets at a glance, color-coded by status and priority.
Note
HelpDesk Companion is an internal IT tool. It requires access to the Pelican State Credit Union network and appropriate Active Directory permissions.

The Main Window

The main window is divided into two areas: the Sidebar on the left and the Content Panel on the right.

Sidebar Controls

ElementDescription

Unlock SelectedUnlocks the account(s) currently selected in the list.
Unlock AllUnlocks every account currently shown in the list after confirmation.

Event Log

The event log at the bottom records every action during your session. Entries are color-coded: green for success, red for errors, blue for informational events. All entries are also saved to a daily CSV on the network share.

Unlocking Accounts

Unlock a Single Account

1
Click the account in the results list to select it.
2
Click Unlock Selected in the sidebar, or right-click the row and choose Unlock Account.
3
Confirm the action in the dialog that appears.
4
The account is unlocked and the event is logged.

Unlock Multiple Accounts

Hold Ctrl and click each account you want to unlock, then click Unlock Selected.

Unlock All Accounts

Click Unlock All in the sidebar. A confirmation dialog asks you to verify before all listed accounts are unlocked.

Warning
Use Unlock All with care. It will unlock every listed account, including ones you may not have reviewed individually.

Password Reset

1
Right-click a user in the results list.
2
Select Reset Password from the context menu.
3
Type a new password and confirm it. Use Generate Strong to auto-generate a secure 16-character password.
4
Leave User must change password at next logon checked (recommended) or uncheck if not required.
5
Click Reset Password.
Password Requirements
Passwords must be at least 16 characters and include at least one uppercase letter, one number, and one special character.

Lockout Source Finder

Use this when a user keeps getting locked out and you need to find which computer or application is causing it.

1
Right-click the locked user in the results list.
2
Select Find Lockout Source.
3
The app searches the PDC Emulator for the most recent lockout event (Event ID 4740).
4
A dialog shows the source computer name and the time of the lockout.
Note
If no event is found, the lockout may have been cleared from the domain controller's security log. Try unlocking and waiting to see if it re-locks.

Group Membership

Right-click any user or computer and select Group Membership. A window opens listing every Active Directory group the account belongs to, including scope and category.

Adding to a Group

1
Type part of the group name in the search box at the bottom of the window.
2
Click Search Groups (or press Enter). Matching groups appear in the results list below.
3
Select the desired group from the results, then click Add to Group.
4
Confirm the action. The membership list refreshes automatically.

Removing from a Group

1
Select the group you want to remove the account from in the membership list.
2
Click the Remove from Selected Group button in the header (highlighted in red).
3
Confirm the action. The list refreshes to reflect the change.
Permissions Required
Adding and removing group members requires you to have sufficient Active Directory delegation rights. Contact your domain administrator if the action fails with an access error.

Printer Manager

The Printer Manager opens as a separate window. Right-click a computer → Manage Printers.

Installing a Printer

1
Click Install Printer.
2
Enter the printer name, select a driver from the dropdown, and enter the printer's IP address.
3
If the driver isn't installed remotely, click Upload Local Driver to select a .INF file. The app copies and installs it remotely.
4
Click Install. A progress dialog tracks the installation.
Driver Deployment
Drivers are copied to C:\Temp\HDC_Drivers on the remote computer and installed via PnPUtil. This may take a minute for large driver packages.

Other Printer Actions

  • Remove Printer — Select the printer, click Remove, confirm.
  • Rename Printer — Right-click → Rename Printer, enter new name.
  • Printer Details — Select and click Printer Details, or double-click the row.
  • Restart Spooler — Stops and restarts the Windows Print Spooler service remotely.

Recent Ticket Feed

Right-click any user with an email address in AD and select Recent Tickets to see their last 8 Freshservice tickets in a card view.

BadgeMeaning

ResolvedSolution has been provided.

Urgent / HighPriority indicators — red for Urgent, amber for High.

Click Open in Freshservice to jump directly to the user's requester profile in the browser.

Requirement
The user must have an email address populated in Active Directory. The Freshservice API key must also be configured in Windows Credential Manager.

Automation Features

Auto-Refresh

Use the dropdown in the sidebar to set how often the locked accounts list refreshes automatically. Options range from 30 seconds to 30 minutes.

Auto-Unlock

Check Auto-Unlock accounts to automatically unlock any locked accounts on a regular schedule (default: every 5 minutes). Useful for overnight shifts or when resolving cached credential issues.

Use With Care
Auto-Unlock may mask underlying lockout causes. All auto-unlock events are logged. Investigate repeated lockouts rather than relying solely on automation.

Viewing Logs

Click View Logs in the sidebar to open the Log Viewer. Browse logs by date, filter by event type, search for a specific username, and export filtered views. Logs are retained for 90 days on the network share.

Troubleshooting

ProblemSolution
List is empty after launchClick Refresh List. If still empty, there may be no locked accounts. Confirm AD connectivity.
Search returns no resultsCheck spelling. The search matches partial names but the account must exist in AD.
Unlock fails with access errorYou may lack sufficient AD permissions. Contact your domain administrator.
Password reset failsEnsure complexity requirements are met (16+ chars, uppercase, number, special). Check event log for details.
Remote computer tools unavailableTarget may be offline, WinRM may not be enabled, or you lack admin rights.
Printer Manager doesn't openVerify PrinterManager.ps1 is in the app folder and PowerShell execution policy allows scripts.
Freshservice actions failVerify connectivity to freshservice.com port 443. Run Set-FSApiKey in PowerShell to re-enter the API key into Credential Manager.
Ticket Feed shows no resultsConfirm the user has an email address in AD and that a Freshservice requester account exists with that email.
Device Info fails on LenovoThis is a known WMI issue with Lenovo firmware. Should be resolved automatically — if it persists, ensure WinRM is enabled and you have admin rights on the target.
